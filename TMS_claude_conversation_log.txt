## Decision 2 — TMS.Shared (netstandard2.0)

### `ApiResponse<T>` Shape

```
ApiResponse<T>
├── bool      Success
├── T?        Data
├── string?   Message
├── List<string> Errors   (empty list, never null)
└── int       StatusCode

Static factory methods:
  Ok(T data, string? message)         → Success=true,  Data=data,    StatusCode=200
  Fail(string message, int code)      → Success=false, Data=default, StatusCode=code
  Fail(List<string> errors, int code) → Success=false, Errors=errors, StatusCode=code
```

Every API response — success or error — is wrapped in this envelope. Controllers return `ApiResponse<TaskItemDto>`, middleware returns `ApiResponse<object>` for error cases. The caller always deserializes the same shape regardless of outcome.

**Why netstandard2.0?**
netstandard2.0 is the broadest compatibility surface in the .NET ecosystem. A project targeting net8.0 can reference a netstandard2.0 library because the .NET 8 runtime fully implements the netstandard2.0 API surface. A hypothetical .NET Framework 4.7.2 project can also reference the same binary because .NET Framework 4.7.2 also implements netstandard2.0. The shared library compiles once and requires zero changes to be consumed by either runtime — no conditional compilation, no multi-targeting. This is the standard pattern for shared contracts and envelope types that may eventually be referenced by client SDKs, shared libraries, or legacy services.

## Decision 5 — Infrastructure Layer

### `TmsDbContext`

Inherits `DbContext`. Single `DbSet<TaskItem> Tasks`.

**`IEntityTypeConfiguration<TaskItem>` configuration (in `TaskItemConfiguration`):**

| Concern | Rule |
|---|---|
| Table name | `Tasks` |
| Primary key | `Id` |
| `Title` | Required, max length 200 |
| `Description` | Optional, max length 2000 |
| `AssignedTo` | Optional, max length 100 |
| `Status` | Stored as int |
| `Priority` | Stored as int |
| `IsDeleted` | Has default value `false` |
| `CreatedDate` | No default at DB level — set by override |
| `ModifiedDate` | No default at DB level — set by override |

No global query filter on `IsDeleted` at the DbContext level. The Application service applies `Where(t => !t.IsDeleted)` explicitly, keeping the filter visible and testable.

### `SaveChangesAsync` Override

Before delegating to `base.SaveChangesAsync()`:

1. Iterate `ChangeTracker.Entries<TaskItem>()`
2. If `EntityState.Added` → set `CreatedDate = DateTime.UtcNow` and `ModifiedDate = DateTime.UtcNow`
3. If `EntityState.Modified` → set `ModifiedDate = DateTime.UtcNow`

This ensures timestamps are never forgotten regardless of which service method persists the entity.

### Migration Plan

Single initial migration named `InitialCreate` generated via:

```
Add-Migration InitialCreate -Project TMS.Infrastructure -StartupProject TMS.API
Update-Database
```

The migration creates the `Tasks` table with all columns as configured. No additional migrations until a schema change is required.

### Seed Strategy — `DataSeeder`

- `DataSeeder` is a static class with a single `SeedAsync(IServiceProvider services)` method.
- Runs **only** when `Environment.GetEnvironmentVariable("SEED_DATA") == "true"`.
- Checks whether any tasks already exist before inserting (idempotent — safe to call on every startup without duplicating data).
- Inserts 10+ `TaskItem` records covering all combinations: every `TaskStatus` (ToDo, InProgress, Done) × every `Priority` (Low, Medium, High, Critical), plus extras to exceed 10 total rows.
- Called from `Program.cs` after `app.Build()` and before `app.Run()`.

### Raw SQL Summary Query

`GetSummaryAsync` bypasses EF entirely, using `DbContext.Database.GetDbConnection()` and a `DbDataReader`:

```sql
SELECT Status, Priority, COUNT(*) AS Count
FROM Tasks
WHERE IsDeleted = 0
GROUP BY Status, Priority
ORDER BY Status, Priority
```

The method opens the connection manually if not already open, executes `DbCommand`, reads each row into a `TaskSummaryDto`, and closes the connection. No EF, no Dapper, no ORM mapping — the requirement for demonstrating raw ADO.NET is met explicitly.

### Cross-Cutting Concerns

**`GlobalExceptionMiddleware`**
- Wraps the entire pipeline in a `try/catch`.
- `NotFoundException` → HTTP 404, `ApiResponse.Fail(ex.Message, 404)`
- `ValidationException` (FluentValidation) → HTTP 400, `ApiResponse.Fail(errors, 400)` where errors is the list of `ValidationFailure.ErrorMessage` strings
- All other exceptions → HTTP 500, `ApiResponse.Fail("An unexpected error occurred.", 500)`
- Uses `ILogger<GlobalExceptionMiddleware>` — logs full exception with stack trace at `Error` level internally.
- In production (`!IsDevelopment`), the 500 message is the generic string above; stack trace is never serialized into the response body.

**`RequestLoggingMiddleware`**
- Starts a `Stopwatch` before calling `next(context)`.
- After the inner pipeline completes, logs: `{Method} {Path} responded {StatusCode} in {ElapsedMs}ms` via `ILogger<RequestLoggingMiddleware>` at `Information` level.

**CORS — `AllowFrontend` policy**
- `WithOrigins("http://localhost:5173")`
- `AllowAnyHeader()`
- `AllowAnyMethod()`
- Named policy applied globally via `app.UseCors("AllowFrontend")`.

**JWT Bearer**
- Configuration sourced exclusively from `appsettings.json` and environment variable overrides — never hard-coded.
- Keys: `Jwt:Issuer`, `Jwt:Audience`, `Jwt:Key`
- Token validation: validates issuer, audience, lifetime, and signing key.
- `[Authorize]` on all task endpoints; `[AllowAnonymous]` on login.

**HTTPS Redirection**
- `app.UseHttpsRedirection()` enforced unconditionally.
- Kestrel configured to listen on `https://localhost:5001` in `launchSettings.json`.

**FluentValidation**
- `builder.Services.AddFluentValidationAutoValidation()` wires automatic model validation.
- `AddValidatorsFromAssemblyContaining<CreateTaskDtoValidator>()` registers all validators in `TMS.Application`.
- Invalid requests are rejected with HTTP 400 before reaching the controller action — no manual `ModelState` checks needed.

**Swagger**
- Enabled in all environments (can be restricted to Development if desired post-approval).
- `SecurityDefinition` added for `Bearer` JWT.

Rewrote Swagger config using standard OpenApiReference / ReferenceType.SecurityScheme API.
